# PRD: AI 기반 KPT 회고 분석 서비스 (KPT Insight)

## 1. 문제 정의 (Problem Statement)
- **휘발성 회고**: 분기별로 작성된 회고 내용이 다음 분기로 이어지지 않고 잊혀짐.
- **분석의 어려움**: 팀원의 수가 많아질수록 공통된 문제점(Problem)을 수동으로 파악하고 개선안(Try)을 도출하는 데 많은 리소스 소요.
- **정성 데이터의 방치**: 팀의 성장 궤적을 확인할 수 있는 정량적 지표 부족.

## 2. 목표 및 핵심 가치 (Goals & Core Values)
- **인사이트 자동 도출**: AI를 활용해 정성적인 회고 데이터에서 핵심 키워드와 액션 아이템 추출.
- **지속 가능한 성장**: 지난 분기의 Try 항목 이행 여부를 확인하여 피드백 루프 완성.
- **접근성 극대화**: PC와 모바일 어디서든 간편하게 회고를 작성하고 분석 리포트 확인.

## 3. 핵심 기능 (Core Features)
- **Responsive KPT Input**: 모바일 환경에 최적화된 KPT 입력 폼.
- **AI Analysis Engine**: Claude 3.5를 활용한 개인별/팀별 통합 분석 리포트 생성.
- **Retrospective Dashboard**: 분기별 팀 성장 지표 및 주요 키워드 시각화.
- **History Tracking**: 과거 회고 내역 조회 및 변화 추이 확인.

## 4. 가설 및 검증 (Hypothesis & Validation)
### 검증 가설
1. AI 분석 리포트를 제공하면 팀원들이 다음 분기 액션 아이템(Try)을 인지하는 속도가 수동 분석 대비 2배 이상 빠를 것이다.
2. 모바일 반응형 웹 제공으로 회고 작성 참여율이 이전 방식 대비 30% 이상 향상될 것이다.

### 성공 지표 (KPI)
- **Primary**: AI 리포트 생성 건수 및 사용자 만족도 점수.
- **Secondary**: Try 항목의 다음 분기 실제 이행률(Completion Rate).

## 5. 사용자 흐름 (User Flow)
1. 로그인 (소셜 로그인)
2. 이번 분기 KPT 작성 (Keep, Problem, Try 입력)
3. AI 분석 요청 버튼 클릭
4. 생성된 리포트 확인 (요약, 공통 이슈, 제안사항)
5. 지난 분기 리포트와 비교 분석

## 6. 기술 검증 계획 (Technical Validation)
### 유닛 테스트 (Unit Testing)
- **Vitest**를 활용하여 핵심 UI 컴포넌트의 렌더링 및 상태 변경 로직 검증.
- KPT 입력 데이터의 유효성 검사 로직 테스트.

### 정적 분석 (Static Analysis)
- **ESLint**: 코드 스타일 및 잠재적 버그 사전 방지.
- **TypeScript**: 엄격한 타입 체크를 통한 런타임 에러 최소화.

### API 통합 테스트
- Claude API 응답 데이터의 마크다운 파싱 및 렌더링 정상 동작 확인.
